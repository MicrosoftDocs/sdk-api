# IWordBreaker::BreakText


## -description


Parses text to identify words and phrases and provides the results to the <a href="https://msdn.microsoft.com/220FCAE5-D22D-45ED-9689-E78C0D8E0BB3">IWordSink</a> and <a href="https://msdn.microsoft.com/9485202D-94D6-4E9E-9C42-502033E85670">IPhraseSink</a> objects.


## -parameters




### -param pTextSource [in]

Type: <b><a href="https://msdn.microsoft.com/library/Bb231254(v=VS.85).aspx">TEXT_SOURCE</a>*</b>

Pointer to a <a href="https://msdn.microsoft.com/library/Bb231254(v=VS.85).aspx">TEXT_SOURCE</a> structure that contains Unicode text.


### -param pWordSink [in]

Type: <b>IWordSink*</b>

Pointer to the <a href="https://msdn.microsoft.com/220FCAE5-D22D-45ED-9689-E78C0D8E0BB3">IWordSink</a> object that receives and handles words generated by this method. <b>NULL</b> indicates that this method should identify phrases only.


### -param pPhraseSink [in]

Type: <b>IPhraseSink*</b>

Pointer to the <a href="https://msdn.microsoft.com/9485202D-94D6-4E9E-9C42-502033E85670">IPhraseSink</a> object that receives and handles phrases generated by this method. <b>NULL</b> indicates that this method should identify individual words, not phrases.

## -remarks



Because word breakers more commonly parse for words than phrases, you should optimize for <i>pPhraseSink</i> = 0. Either <i>pWordSink</i> or <i>pPhraseSink</i> can be <b>NULL</b>, but not both.

The <a href="https://msdn.microsoft.com/220FCAE5-D22D-45ED-9689-E78C0D8E0BB3">IWordSink</a> object holds the words and their alternative forms for the word breaker. Alternative forms of words, if they exist, are put in the <b>IWordSink</b> object first, by using the <b>WordSink::PutAltWord</b> method, and the root word is added last, by using the <b>WordSink::PutWord</b> method.

Use <b>pfnFillTextBuffer</b>, the function pointer element in the <a href="https://msdn.microsoft.com/library/Bb231254(v=VS.85).aspx">TEXT_SOURCE</a> structure, to replenish the source text. The <b>IWordBreaker::BreakText</b> method must handle all <b>pfnFillTextBuffer</b> return values. If an error occurs, finish processing the text in the buffer before handling the error.
